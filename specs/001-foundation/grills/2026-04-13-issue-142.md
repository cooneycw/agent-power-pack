## Q1: Are you sure `.agents/skills/<name>/SKILL.md` is the current Codex-native skill discovery path?

The issue cites OpenAI Codex skills docs and local `codex-cli 0.114.0` verification from 2026-04-12. The evidence is strong enough to proceed. The acceptance criteria also include verifying with a real Codex session post-merge.

## Q2: Should the Codex SKILL.md use identical frontmatter to Claude's SKILL.md?

No. Codex uses SKILL.md but has its own frontmatter expectations. The approach preserves Codex-specific metadata (triggers, mcp_tools) that were previously in openai.yaml, combined with the manifest source metadata.

## Q3: What about the prompts directory? Is that still relevant?

No. The old layout split agent definition (openai.yaml) from prompt content (prompts/). In the SKILL.md model, everything goes in a single file — same as Claude's adapter. The prompts directory goes away.

## Q4: Does removing `.codex/` as the base directory break user-mode installs?

For project mode, we change from `.codex/skills/` to `.agents/skills/` (repo-scoped). For user mode, skills go to `~/.agents/skills/`. The MCP config.toml merge is separate and stays at `~/.codex/config.toml`.

## Q5: Will changing the golden fixture directory structure break other tests?

No. The golden fixtures are self-contained under `tests/integration/golden/codex/`. Only the contents change, not the test lookup path.
